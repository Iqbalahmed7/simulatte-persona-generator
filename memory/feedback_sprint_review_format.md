---
name: Sprint review output format
description: After each sprint review, publish only a concise ratings table — not the full review narrative
type: feedback
---

After completing a sprint review, publish only the ratings table to the user:

| Engineer | Spec | Completeness | Quality | Criteria | Total |

One row per engineer. Idle engineers get dashes.

**Why:** User wants to be aware of performance without reading the full review. The full review goes into the archive file only.

**How to apply:** Every sprint. Table immediately after review is complete. No preamble, no detailed breakdown unless user asks.
